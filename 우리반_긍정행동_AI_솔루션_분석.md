## 3. 이용약관 및 운영정책 페이지

본 사이트는 가입 단계에서 다음 5개 조항에 대한 **필수 동의**를 요구한다.

### 3.1 개인정보 보호 (필수)

1. **학생 실명 입력 금지** — "OO학생" 등 익명 형태로만 입력
2. **민감정보 최소 입력** — 주민번호·상세주소·불필요한 연락처 금지
3. **관리 책임의 귀속** — 관련 법령 및 기관 지침 준수, 입력 정보 관리 책임은 **이용자(교사)**에게 있음

### 3.2 AI 결과물 성격 및 책임 (필수)

- AI 결과물은 오류 가능성이 있으며 **참고용**
- 업무 적용 전 반드시 **교사가 검토·수정·확인** 필요
- **최종 책임은 교사**에게 귀속

### 3.3 이용범위 및 금지행위 (필수)

허용: 교육 목적

금지:
- 무단 복제/배포/수정/2차 가공 (화면, 산출물 양식, 문구, 코드, 구조 등 일체)
- 역공학(리버스 엔지니어링), 소스·로직·프롬프트 탈취·분석, 보안장치 우회
- 유사 서비스 제작·판매·재배포 (유·무료 포함)

위반 시 서비스 이용 제한 및 민·형사상 책임 발생 가능.

### 3.4 권리 귀속 (필수)

- 저작권 및 관련 권리는 **개발자 충북도교육청 초등특수교사 김대한**에게 귀속

### 3.5 동의 기록 저장 (필수)

분쟁 예방 및 약관 준수 확인을 위해 동의 메타데이터 저장 (위 2.1의 수집 항목 참조).

## 4. 학급 차원 긍정적 행동지원 (Tier 1) 페이지

### 4.1 학급 강화 시스템

**🏆 학급 목표: 모두가 행복한 교실**

| 기능 | 설명 |
|------|------|
| 목표 설정 | 교사가 학급 공동 목표와 포인트 기준선을 설정 |
| 진행 게이지 | 0/100 포인트 형태의 시각적 누적 표시 |
| 목표 달성 버튼 | `🎉 목표 달성!` 클릭 시 보상 단계로 이동 |
| 초기화 | 새 주기 시작 |

**핵심 원칙: 4:1 비율(긍정:부정)**
- 바람직한 행동 대비 재지도 비율을 4:1 이상으로 유지
- 길라잡이(안내 문서) 팝업 제공

### 4.2 재지도가 필요한 행동 (약속 상기)

학급 규칙을 벗어난 행동 발생 시 **처벌이 아닌 약속 상기** 방식으로 접근하는 영역. 규칙을 다시 안내하고 자기조절 기회 제공.

### 4.3 AI 학급 운영 코칭 (서울형 PBS)

**입력 → 출력 흐름**
1. 2024 서울시교육청 PBS 가이드북(초등·중등) 참조
2. 교사의 누적 데이터(학생 ABC, QABF, BIP 실행 기록 등) 결합
3. 학교급(초등/중등)에 따른 맞춤 코칭 생성

**버튼**
- `💡 AI 코칭 받기` — 학급 운영 전반에 대한 AI 제안 요청

### 4.4 학급 보상 설정 모달

(별도 모달로 분리)
- 보상 항목 이름·설명·필요 포인트 설정
- 취소 / 저장

## 5. 학생 관찰 및 이해 페이지 (1단계)

### 5.1 학생 기초 정보 블록

**구성**
- 학생 선택 드롭다운 — 상단에서 선택된 대상 학생 정보 표시
- "학생을 선택해주세요." 안내 메시지 (미선택 시)
- 프로필 수정 버튼 — 모달 열림
- `💡 선호/강화물 탐색 (RAISD)` — **RAISD**(Reinforcer Assessment for Individuals with Severe Disabilities) 기반 선호도 조사 도구

### 5.2 보조 도구 버튼

| 버튼 | 기능 |
|------|------|
| 📋 문제행동 우선순위 체크리스트 | 총점 0/36 형태의 4점 척도(0=전혀 아님 ~ 4=매우 그렇다)로 중재 우선순위 결정 |
| 🪄 조작적 정의 도우미 (Dead Man's Test) | 모호한 서술을 관찰 가능한 행동으로 변환 |

**Dead Man's Test**: "죽은 사람도 할 수 있으면 행동이 아니다"는 원칙. 예) "수업을 방해하지 않는다"(부적절) → "수업 시간에 자리에 앉아 과제를 수행한다"(적절).

### 5.3 ABC 행동 관찰 기록

**입력 필드**
- A (Antecedent, 선행사건)
- B (Behavior, 행동)
- C (Consequence, 결과)
- 날짜·시간·장소·관찰자 (추정)
- 관찰 기록 저장 버튼

**📋 작성 예시 보기 (모달)**

| 구분 | ❌ 나쁜 예시 (주관적 해석) | ✅ 좋은 예시 (관찰 가능한 사실) |
|------|-----------------------------|------------------------------------|
| A | 기분이 안 좋아 보일 때 | 수학 익힘책 15쪽을 풀라고 지시했을 때 |
| B | 반항적으로 행동했다 | "싫어!"라고 소리치며 책을 바닥에 던졌다 |
| C | 혼냈다 | 교사가 다가가 "책을 주우세요"라고 지시했으나 거부함 |

출처: 국립특수교육원

### 5.4 누적된 ABC 데이터 영역

- 시간 순으로 저장된 기록을 리스트로 표시
- 빈 상태: "저장된 기록이 없습니다."
- 후속 FBA/QABF 분석을 위한 원천 데이터로 활용

### 5.5 새 학생 추가 모달

**구성**
- 익명 ID 입력
- 기본 정보 (학년 계열, 주요 장애 영역 등 추정)
- **🔒 AI 전송용 요약 (비식별) 필드** — 이 필드 내용만 AI로 전송
- 주의 문구: "이름·학년·실명·민감정보를 넣지 마세요."

## 6. AI 기능 평가 (FBA/QABF) 페이지 (2단계)

### 6.1 분석 영역 헤더

- `💾 분석 결과 수시 저장` — 중간 결과 저장
- `🤖 AI 행동 기능 분석 요청` — 메인 실행 버튼

### 6.2 분석할 ABC 데이터 선택

- 5단계에서 누적한 ABC 기록 중 복수 선택 가능
- 여러 기록 선택 시 AI가 **누적된 데이터 패턴을 심층 분석**하여 행동의 기능 추론
- 빈 상태: "저장된 기록이 없습니다."

### 6.3 📋 QABF 척도 (Questions About Behavioral Function)

**구성**
- 25문항 체크리스트 (펼치기/접기 UI)
- 4점 척도: 0 = 전혀 아님 / 1 = 가끔 / 2 = 자주 / 3 = 매우 자주
- 5가지 기능 범주로 자동 분류, 각 기능별 **0-15점** 산출
  - **관심** (Attention)
  - **회피** (Escape)
  - **획득** (Tangible)
  - **자동 강화 / 감각** (Automatic / Sensory)
  - (QABF 원안은 5요인이나 화면 안내는 4기능 표기)

**활용**
- 회의록·IEP 보고서에 척도 점수 첨부
- AI 분석 결과와 교차 검증

### 6.4 AI 분석 결과 출력 영역

**처리 중 메시지**
> 선생님의 데이터와 서울 PBS 가이드북을 기반으로 분석 중입니다...

**빈 상태**
> 분석할 상황 기록을 선택해 주세요. AI가 관심, 회피, 획득, 감각(자동) 기능을 분석합니다.

**정상 출력**
각 기능별 추론 근거(ABC 패턴, 선행·결과 요인), QABF 점수와의 일치도, 중재 방향 제언.

## 7. AI 긍정적 행동중재계획 (BIP) 페이지 (3단계)

### 7.1 상단 기능 버튼

| 버튼 | 설명 |
|------|------|
| 📜 AI 중재안 생성 | 예방-교수-강화 3요소 기반 BIP 초안 생성 |
| ✍️ 행동계약서(중등) | 중등 가이드북 권장 양식의 계약서 모달 |
| ✉️ 가정 연계 통신문 | 학부모에게 전달할 중재 안내 문서 생성 |
| 📄 인쇄/PDF | 생성 결과물 인쇄·저장 |

### 7.2 🎯 목표 행동(대체 행동) 설정

**입력 요소**
- 대체 행동 선택/입력 — 문제행동을 대신할 바람직한 행동
- **FCT(Functional Communication Training)** 기술 선택
  - 예: "쉬고 싶을 때 '쉬고 싶어요' 카드 들기"
- 성공 기준: "N회 미만/이상" 형태로 발생 빈도 기준 설정
- 저장 버튼

### 7.3 AI 중재안 출력 영역

**빈 상태**
> 목표 행동을 선택하고 중재안을 생성하세요. 가이드북에 기반한 예방-교수-강화 전략을 제공합니다.

**생성 결과 구조 (추정)**
1. **예방 전략 (Antecedent Strategies)** — 환경 조정, 선택권 제공, 시각적 지원
2. **교수 전략 (Teaching Strategies)** — 대체 기술 교수, 사회성 기술, FCT
3. **강화 전략 (Reinforcement Strategies)** — 차별 강화(DRA/DRO), 토큰 경제
4. **반응/위기 절차 (Response Procedures)** — 문제행동 발생 시 대응

### 7.4 ✍️ 행동 계약서 (Behavior Contract) 모달

**양식 필드**
- 나(학생)의 약속 (목표 행동)
- 성공 기준 (기간/빈도)
- 선생님의 약속 (보상/강화)
- 계약 기간 (시작일 ~ 종료일)
- 취소 / 🖨️ 계약서 생성 및 인쇄

**대상**: 중등 가이드북 권장 (초등에도 활용 가능)

### 7.5 ✉️ 가정 연계 통신문

가정과의 협력을 위한 문서. 학생의 목표 행동, 가정 내 강화 방법, 진행 상황 공유를 위한 표준 서식.

## 13. 위기행동 관리 및 대처 시뮬레이션 페이지

### 13.1 🚨 위기행동 대처 시뮬레이션 요청 (AI)

**안내 문구**
> 학생의 위기 행동 상황을 구체적으로 묘사하면, 가이드북에 기반한 단계별 대응 전략을 AI가 제안합니다.

**주요 버튼**
- `🚨 위기행동 단계별(7단계) 대처 방안 작성`

### 13.2 위기 행동 7단계 및 대응 전략

| 단계 | 명칭 | 영문 | 대응 전략 |
|------|------|------|-----------|
| 1 | 안정 | Calm | 예방 전략 유지, 칭찬 및 강화 |
| 2 | 전조 | Trigger | 자극 요소 제거, 관심 전환 유도 |
| 3 | 흥분 | Agitation | 공감적 경청, 공간 제공, 선택권 부여 |
| 4 | 가속 | Acceleration | 지시 최소화, 안전 거리 확보 |
| 5 | 고조 | Peak / 위기 | 안전 확보 최우선, 주변 학생 대피, 매뉴얼에 따른 위기관리 |
| 6 | 탈고조 | De-escalation | 간섭 최소화, 독립적 공간 제공 |
| 7 | 회복 | Recovery | 쉬운 과제로 복귀 유도, 사후 협의 |

> 이는 Colvin & Sugai(1989)의 **Acting-Out Cycle 7단계 모델**에 기반한 것으로, 특수교육 위기관리의 국제 표준.

### 13.3 ⚠️ 신체적 개입 5대 원칙 (가이드북 부록)

| 원칙 | 설명 |
|------|------|
| 안전 최우선 | 학생과 주변인의 안전 확보가 목적 |
| 최후의 수단 | 비신체적 중재가 효과가 없을 때만 |
| 최소한의 힘 | 필요한 범위 내 최소한, 압박 금지 |
| 일시적 사용 | 상황 안정 시 즉시 해제 |
| 기록 및 보고 | 개입 후 반드시 기록하고 보호자 통보 |

### 13.4 💡 분노 조절 — 그라운딩 5-4-3-2-1 기법

| 감각 | 수 | 활동 |
|------|-----|------|
| 👀 시각 | 5가지 | 눈에 보이는 것 말하기 |
| 🖐️ 촉각 | 4가지 | 몸에 느껴지는 감각 말하기 |
| 👂 청각 | 3가지 | 들리는 소리 말하기 |
| 👃 후각 | 2가지 | 냄새 맡아보기 |
| 👅 미각 | 1가지 | 맛 느껴보기 (또는 좋은 점 하나 말하기) |

**효과**: 해리·불안·공황 상황에서 현재 감각에 집중시킴으로써 자율신경계를 안정시키는 **근거 기반 정서 조절 기법**.

| Single Subject Design | 평가 | Phase 전환 시각 분석 |
| Acting-Out Cycle 7단계 | 위기관리 | Colvin & Sugai 모델 |
| 5-4-3-2-1 Grounding | 정서조절 | 감각 기반 안정화 |
| 3-Tier Model | 체계 | 보편·소그룹·개별 차등 지원 |
| 4:1 Ratio | 강화 | 긍정:부정 상호작용 황금률 |

### 15.2 교육학적·윤리적 강점

- **서울시교육청 공식 가이드북** 기반 → 행정적 신뢰성
- **교권 보호 및 교사 회복 지원** 포함 → 현장 친화적
- **개인정보 비식별화 강제** → 학생 보호·법령 준수
- **교사의 최종 책임 명시** → AI 과신 방지
- **초·중등 통합 지원** → 범용성

### 15.3 추천 활용 시나리오 (학기 흐름 기준)
